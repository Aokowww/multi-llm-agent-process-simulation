\section{Conclusion}\label{conclusion}

This project designed and evaluated a resource-centric LLM-augmented
multi-agent process simulation artifact. The main contribution is a
constrained architecture in which LLM-style local decisions remain
inside event-log-derived action spaces, produce
process-mining-compatible event logs, and add reasoning and handover
traces.

The empirical evaluation shows dimension-specific trade-offs. On
AcademicCredentials, the log-derived agent-profile baseline performs
best on formal control-flow n-grams and absolute/case-arrival timing
distances. The LLM-agent proxy remains close on control-flow measures
and performs best on workforce distance, but it is not the best overall
simulator and underperforms on some temporal distribution metrics. The
LoanApp robustness check leads to the same bounded interpretation. The
proxy can improve workforce distance and some high-load what-if
responses, but the advantage does not hold under every stress condition.
Overall, log-derived resource profiles and local priors can constrain
LLM-style agent decisions sufficiently to produce valid event logs and
interpretable traces. Claims about accuracy should continue to be
validated across both BPS quality and agent-behaviour dimensions.
